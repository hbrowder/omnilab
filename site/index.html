Why OmniLab

Six things every other browser-based lab tool gets wrong, in our opinion.

🔒

Security stacks that actually work

Real Linux kernels means real Suricata, Wazuh, and Kali tooling. IDS evasion against a containerized kernel isn't IDS evasion.

🛠

DevOps without the cloud bill

Multi-node Kubernetes, Terraform-managed VMs, CI runners on isolated networks — all on hardware you already own.

🤖

AI/ML labs, GPU-aware

Spin Ollama next to Jupyter, mount a passthrough GPU, run RAG experiments. Snapshot the lab and ship it to a teammate.

🌐

Networking without the licence dance

Mix Cisco, Juniper, VyOS, and FRR in the same topology. Configure link quality (delay, jitter, loss, bandwidth) per edge.

📦

Portable lab format

Every lab exports to a portable JSON file. Hand it to a student, drop it in a repo, version it in git. Re-import anywhere.

⚡

Browser, CLI, or API

Drag-and-drop topology editor for designers. Full CLI (omnilab lab deploy …) for scripters. REST API for everyone else.
